Java Collections - Cuộc Phiêu Lưu Với Dữ Liệu

6 tháng 10, 2025 Nguyễn Tiến Dũng

Khi Array không còn đủ

Sau khi làm quen với array, tôi nghĩ mình đã master việc lưu trữ dữ liệu. Đến khi làm project quản lý sinh viên với 1000+ records, tôi mới hiểu tại sao cần Collections. Array cố định size, khó add/remove, còn Collections thì linh hoạt hơn nhiều!

ArrayList vs LinkedList - Cuộc chiến đầu tiên

Mentor hỏi: "Em dùng ArrayList hay LinkedList?" Tôi trả lời bừa: "ArrayList". Hắn cười và bảo làm bài test. Kết quả: ArrayList nhanh hơn khi access theo index, LinkedList nhanh hơn khi insert/delete ở giữa. Bài học đầu tiên: hiểu performance characteristics!

HashSet - Lần đầu gặp "unique"

Dự án đầu tiên cần lưu danh sách email unique. Tôi dùng ArrayList và viết loop check duplicate. Code dài và chậm! Mentor show cho tôi HashSet - chỉ cần add(), nó tự động loại bỏ duplicate. Mind blown!

HashMap - Cách mạng trong tư duy

Trước khi biết HashMap, tôi loop qua List để tìm object theo ID. O(n) complexity! Khi biết HashMap với key-value, tìm kiếm thành O(1). Từ đó tôi hiểu tại sao người ta nói "choose right data structure".

Những sai lầm kinh điển với Collections

  • Dùng Vector thay ArrayList: Không biết Vector synchronized và chậm hơn
  • Quên override equals/hashCode: HashSet, HashMap không hoạt động đúng
  • Modify collection khi iterate: ConcurrentModificationException xuất hiện
  • Dùng sai Collection: TreeSet cho performance nhưng cần Comparable

Stream API - Game changer

Java 8 ra đời với Stream API, tôi như được sinh ra lần thứ hai! Thay vì viết loop dài để filter, map, reduce, giờ chỉ cần chain operations. Code ngắn hơn, đọc như English!

Hiệu năng - Bài học đắt giá

Dự án production đầu tiên của tôi bị lag vì sai chọn Collection. LinkedList với 10k elements, access by index = disaster! Từ đó tôi luôn benchmark và profile trước khi chọn data structure.

Tips chọn Collection phù hợp

  • ArrayList: Khi cần access by index và ít insert/delete
  • LinkedList: Khi thường xuyên insert/delete ở giữa
  • HashSet: Cần unique elements và không care thứ tự
  • TreeSet: Cần unique + sorted elements
  • HashMap: Key-value lookup nhanh
  • TreeMap: Key-value + sorted by key

Những Khó Khăn Khi Học Java Collections

Những thử thách mà developer gặp phải khi học Java Collections Framework:

Chọn Collection Type Phù Hợp

Vấn đề: Không biết khi nào dùng ArrayList, LinkedList, HashSet, TreeSet... dẫn đến performance kém.

Giải pháp: Học time complexity của từng operation. ArrayList cho random access, LinkedList cho insertion/deletion.

equals() và hashCode() Contract

Vấn đề: HashMap, HashSet không hoạt động đúng khi custom objects không override equals/hashCode properly.

Giải pháp: Luôn override cả equals() và hashCode() cùng nhau. Sử dụng IDE generate hoặc Objects.hash().

ConcurrentModificationException

Vấn đề: Modify collection (add/remove) khi đang iterate bằng enhanced for loop hoặc iterator.

Giải pháp: Sử dụng Iterator.remove() hoặc collect elements cần remove trước, sau đó removeAll().

Generics và Type Safety

Vấn đề: Raw types, wildcard generics (?), type erasure gây confusion và ClassCastException.

Giải pháp: Luôn sử dụng generics, hiểu bounded wildcards (? extends, ? super), tránh raw types.

Performance Characteristics

Vấn đề: Không hiểu time complexity: ArrayList.remove(0) là O(n), HashMap resize overhead.

Giải pháp: Học Big O notation, benchmark performance, hiểu internal implementation của từng Collection.

Tips học hiệu quả:

  • Visualize data structures: Hiểu internal implementation qua diagrams
  • Benchmark performance: Test với data size khác nhau để hiểu complexity
  • Practice with real scenarios: Implement cache, shopping cart, user management
  • Read source code: JDK source code của ArrayList, HashMap rất educational

Kết Luận

Collections không chỉ là công cụ để lưu trữ dữ liệu, mà còn ảnh hưởng lớn đến hiệu suất và chất lượng code. Hiểu rõ từng loại Collection và biết khi nào dùng cái gì là kỹ năng quan trọng của mọi Java developer!
